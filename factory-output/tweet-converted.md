# 新蓝人计划（2）｜从“抄作业”到“有体系”：把大佬 SKILL 变成你的专属工作流

上篇文章里我说过两句话：

- 先搭台，再唱戏
- 先完成，再完美

这一篇，我继续做一件更实操的事：演示怎么把各路大佬的 SKILL，变成你自己的专属 SKILL。

我一直觉得，最好的学习方式就是模仿。

这时候可能有人会说：这不就是“复制别人的 SKILL”吗？

如果你已经很熟悉这套操作，那你确实不是这篇文章的目标读者。我这篇文章就是写给 0 基础新手的入门演示，有些步骤会故意写得很直白，甚至“笨一点”，因为对新手来说，真正缺的往往不是概念，而是一个看得懂、照着就能做的过程。

对从没系统用过 AI、也没有编程基础的人来说，很多时候真的就差一次“明明白白、实实在在”的演示。看懂一次，路就通了。

我会用宝玉老师的 SKILL 仓库做例子，演示如何把他的 SKILL 改造成你自己的版本。后续你还可以按自己的实际流程继续定制。

## 第一步：先“选妃”——挑一个适合你的 SKILL

先打开宝玉老师 SKILL 仓库的 GitHub 页面。你如果还不熟 GitHub，也没关系，可以先把它理解成一个“放项目文件的网盘”。

![](https://pbs.twimg.com/media/HBqtH2WaEAArs45.jpg)
![](https://pbs.twimg.com/media/HBqtJy9bIAAxH_r.jpg)

进去后会看到很多目录名，看不懂很正常。你可以直接把浏览器地址发给 Codex，让它帮你列出里面有哪些 SKILL、分别做什么。

![](https://pbs.twimg.com/media/HBqs7zwasAAXs7G.jpg)

## 第二步：明确需求，直接点名你要的 SKILL

比如这次我想要“Markdown 格式化”的能力，那我就直接选 baoyu-format-markdown，并告诉 Codex：

- 我想用这个 skill
- 请自动适配我的工作流
- 同时给出优化建议并执行

建议提示词：让 Codex 把该 SKILL 自动适应你的工作流，提出优化建议，并且执行。

![](https://pbs.twimg.com/media/HBqsyFjaUAAdSFz.jpg)
![](https://pbs.twimg.com/media/HBqs_tIb0AA0oOM.jpg)
![](https://pbs.twimg.com/media/HBqtFrGbgAUpZeW.jpg)

## 第三步：重启后进入“自动执行”状态

完成改造后，重启 Codex。之后你再按自己的写作工作流跑流程，Codex 就会在对应步骤里自动执行这个 SKILL。

## 最后想说：模仿不是终点，适配才是

宝玉老师的 SKILL 已经覆盖了大多数人的需求，这点非常强。

但“强”不等于“完全适配你自己”。

所以不要生搬硬套。更实用的做法是：让 AI 基于你的工作流提出适配建议，然后把这些建议真正嵌进你的流程里。

这样你得到的，不是“别人的工具箱”，而是“你自己的生产系统”。
